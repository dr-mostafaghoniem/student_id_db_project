# نظام إدارة طباعة بطاقات هوية الطلاب (Student ID Print Manager)

مرحباً بك! هذا المشروع عبارة عن نظام متكامل ومصمم بأسلوب احترافي وعصري لإدارة طباعة بطاقات الهوية (ID) للطلاب. النظام يلبي جميع متطلباتك بدقة متناهية:
1. **قاعدة بيانات SQLite في ملف واحد**: يتم تخزين كل شيء محلياً في ملف واحد وهو `students_cards.db`.
2. **سهولة النقل والتنقل**: يمكنك نسخ مجلد المشروع بالكامل أو ملف قاعدة البيانات ونقله إلى أي جهاز آخر عبر فلاشة (USB) أو بالبريد الإلكتروني، وسيعمل فوراً دون فقدان أي بيانات.
3. **تقييد عدد الطباعة ببطاقتين فقط**: تفرض قاعدة البيانات قيوداً صارمة تمنع إدخال أو طباعة أكثر من بطاقتين (البطاقة 1 والبطاقة 2) لكل طالب.

---

## 🚀 هندسة التشغيل المزدوجة الذكية (Smart Dual-Mode)

لضمان عمل البرنامج على أي جهاز كمبيوتر، تم تصميم النظام ليعمل بنمطين ذكيين بشكل تلقائي:

### 1. نمط الخادم المحلي المباشر (Server API Mode)
إذا كان جهازك يحتوي على بيئة تشغيل **Python**، يمكنك تشغيل الخادم المحلي. سيقوم البرنامج بالربط المباشر مع ملف قاعدة البيانات `students_cards.db` عبر مكتبة SQLite الرسمية.

### 2. نمط المتصفح المستقل (Stand-alone Offline Mode)
**أفضل ميزة للنقل السهل!** إذا لم تكن لغات البرمجة (مثل بايثون) مثبتة على الجهاز الآخر، يمكنك ببساطة فتح ملف `templates/index.html` في أي متصفح ويب (Chrome, Edge, Firefox) بالنقرتين! 
سيكتشف البرنامج غياب الخادم تلقائياً ويتحول إلى **نمط قاعدة البيانات المحلية الافتراضية** (المحاكاة الكاملة لـ SQLite داخل المتصفح)، حيث يمكنك إضافة الطلاب وطباعة الهويات مع تطبيق نفس القيود والتحقق وحفظ البيانات تلقائياً، مع توفير أزرار **لتصدير قاعدة البيانات كملف احتياطي** ونقلها واستيرادها على أي جهاز آخر بلمسة زر واحدة!

## 📊 هيكل الجداول والقيود في قاعدة البيانات (SQLite Schema)

قاعدة البيانات مبنية من جدولين مرتبطين بعلاقة (One-to-Many):

### 1. جدول الطلاب (`students`)
يخزن بيانات الطلاب برقم أكاديمي فريد يمنع التكرار:
- `id`: معرف تلقائي فريد.
- `student_name`: اسم الطالب بالكامل.
- `academic_id`: الرقم الأكاديمي (Unique - يمنع تسجيل طالبين بنفس الرقم).
- `department`: القسم الأكاديمي.
- `created_at`: تاريخ التسجيل.

### 2. جدول عمليات الطباعة (`id_prints`)
يسجل كل بطاقة تُطبع وتاريخها مع القيود البرمجية التالية:
- `id`: معرف تلقائي.
- `student_id`: معرف الطالب (مفتاح خارجي مرتبط بجدول الطلاب وعند حذف الطالب يُحذف سجله تلقائياً `ON DELETE CASCADE`).
- `print_number`: رقم البطاقة المطبوعة (قيد تحقق `CHECK` يضمن أن القيمة هي **1** أو **2** فقط).
- `print_date`: تاريخ ووقت الطباعة تلقائياً.
- `notes`: ملاحظات اختيارية (مثل: بدل تالف، بدل فاقد).

> [!IMPORTANT]
> **قيد لمنع تجاوز بطاقتين**: تم تعيين قيد فريد ثنائي `UNIQUE(student_id, print_number)`. هذا يعني أنه لا يمكن للطالب الواحد أن يمتلك عمليتي طباعة للبطاقة رقم 1، ولا عمليتي طباعة للبطاقة رقم 2. وبالتالي يستحيل برمجياً إدخال أكثر من بطاقتين للطالب الواحد!

## 🛠️ كيف تبدأ بتشغيل واستخدام النظام؟

### الطريقة الأولى: التشغيل الفوري والسهل (نمط المتصفح المستقل)
1. اذهب للمجلد: `C:\Users\Administrator\.gemini\antigravity\scratch\student_id_db_project\templates`.
2. انقر نقراً مزدوجاً على الملف `index.html` لفتحه في أي متصفح.
3. سيعمل النظام فوراً بواجهة نيون وتصميم زجاجي رائع!
4. عند إجراء تعديلات، انقر على **تصدير البيانات** في القائمة اليمنى لتحميل ملف النسخة الاحتياطية. ونقله لجهاز آخر، ثم اضغط **استيراد** لرفع الملف وتحديث البيانات بلحظة!

### الطريقة الثانية: تشغيل خادم الويب المحلي (Flask Server)
*إذا كان لديك Python مثبت على الجهاز:*
1. افتح نافذة الأوامر (CMD أو Terminal) في مجلد المشروع.
2. قم بتثبيت Flask عن طريق الأمر:
   ```bash
   pip install flask
   ```
3. قم بتشغيل الخادم المحلي بالأمر:
   ```bash
   python app.py
   ```
4. افتح المتصفح واذهب إلى الرابط: `http://127.0.0.1:5000` للتمتع بالنظام متصلاً بقاعدة البيانات SQLite المباشرة.
